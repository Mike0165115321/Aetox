# desktop/ ‚Äî Wails GUI front end

> Module map: [ARCHITECTURE.md ¬ß4.2](../../ARCHITECTURE.md) ¬∑ Deep dives: [desktop-app](../docs/architecture/desktop-app-2026-07-22.md) ¬∑ [browser-security](../docs/architecture/browser-security-2026-07-21.md) ¬∑ [tesseract bundling](../docs/architecture/tesseract-ocr-bundling-2026-07-22.md) ¬∑ Tests: `TEST-REPORT.md` Module 5

**What it is:** the Wails v2 + Svelte 5 desktop app ‚Äî chat, tabbed workbench (Review/Terminal/Files/Browser), SQLite session persistence, and an agent-driven native browser. Embeds the same engine as the CLI (`internal/app.NewApp` + `RunOnce`); everything else here is surface.

## Dev commands

- `wails dev` (or [wails-dev.bat](wails-dev.bat)) ‚Äî live dev, frontend hot reload; browser dev server at `http://localhost:34115`.
- `wails build` ‚Äî production `desktop.exe`.
- `go test ./...` here + `npx svelte-check` in `frontend/` ‚Äî run both before calling a change done.

## Go side (package main)

| File | Role |
|---|---|
| [app.go](app.go) | Wails-bound `App` (distinct from `internal/app.App`). `bootstrapFromConfig` wires engine + workbench tools + MCP; provider/model/approval switching; project tree/file IO; chat attachments; `CancelTurn` (Stop button ‚Äî the brake for the unbounded tool loop); emits `agent:status` + `agent:tool` events to the frontend. |
| [sessions.go](sessions.go) / [db.go](db.go) | Per-project session persistence (SQLite + FTS5 search), transcript ‚Üî `model.Message` restore. |
| [browser.go](browser.go) | Native WebView2 tab host via raw Win32 syscalls ‚Äî Windows-only. Read [browser-security](../docs/architecture/browser-security-2026-07-21.md) **before** touching `onMessage`/`metaScript`/`textScript`. |
| [workbench.go](workbench.go) | `browser_open/read/click/type` as `skill.Tool` (agent-facing; ref-tagging pattern) ‚Äî registered `SourceExternal`. |
| [terminal.go](terminal.go) | User-facing terminal pane (ConPTY) ‚Äî independent of the agent's `shell` tool. |
| [main.go](main.go) | Wails bootstrap. |

## Frontend (`frontend/src/`)

- [App.svelte](frontend/src/App.svelte) ‚Äî layout, panel resize/collapse, Wails event subscriptions (`agent:status`, `agent:tool`), file drop.
- `lib/stores/cockpit.svelte.ts` ‚Äî the single reactive state tree (chat, tool timeline steps, model info); `lib/stores/workbench.svelte.ts` ‚Äî workbench tabs.
- `lib/Chat.svelte` ‚Äî chat + live tool timeline + Stop button; `lib/workbench/` ‚Äî Review/Files/Browser panes; `lib/locales/{th,en}.ts` ‚Äî i18n (add keys to **both**).
- Generated Wails bindings live in `frontend/wailsjs/` ‚Äî regenerated by `wails dev`; if you add a Go method mid-session, mirror it there by hand or rerun `wails dev`.

## Rules of thumb

- Engine behavior does not belong here ‚Äî if it isn't presentation, persistence, or a native-window concern, put it in `internal/*`.
- `wailsruntime.EventsEmit` requires a real Wails context ‚Äî code paths that tests reach must guard `a.ctx != nil` (see `TEST-REPORT.md` on what's structurally untestable).
