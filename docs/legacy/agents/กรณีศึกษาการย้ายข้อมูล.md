User: "ดึงบทความจาก blog.example.com ทั้งหมด ไปโพสต์ที่ mysite.com"

🔄 Workflow:

1. [Planning] MainAgent วางแผน:
   • ขั้นตอนที่ 1: ดึงลิงก์บทความทั้งหมดจาก blog.example.com/sitemap
   • ขั้นตอนที่ 2: สำหรับแต่ละลิงก์ → ดึงเนื้อหา → สรุป → เก็บลง Memory
   • ขั้นตอนที่ 3: แปลงรูปแบบ → โพสต์ไป mysite.com API

2. [Execution - Step 1]
   • เรียก Tool: `WebScraper` ดึง sitemap
   • ผลลัพธ์: [".../post-1", ".../post-2", ...] (50 ลิงก์)
   • เก็บลง WorkingMemory: "พบ 50 บทความ"

3. [Execution - Step 2] (วนลูป)
   • สำหรับแต่ละลิงก์:
     - ดึงเนื้อหา → แบ่งเป็น chunks → เก็บลง Vector DB
     - สรุปสาระสำคัญ → เก็บเป็น MemoryChunk
     - อัปเดต task_context: "ประมวลผลแล้ว 10/50"
   • ✅ โมเดลเห็นแค่ "สรุป" ของแต่ละบทความ ไม่ใช่เนื้อหาเต็ม → ประหยัดบริบท

4. [Execution - Step 3]
   • เมื่อจะโพสต์: ค้นหาจาก Vector DB ด้วยคำสำคัญ "เนื้อหาบทความ"
   • ดึงเฉพาะบทความที่เกี่ยวข้องกับโพสต์ปัจจุบัน
   • ส่งให้โมเดลเขียนโค้ดแปลงรูปแบบ → โพสต์

✅ ผล: โมเดล 8B ทำงานได้เหมือนเห็นเนื้อหาทั้งหมด 
   โดยไม่ต้องโหลด 50 บทความพร้อมกันในบริบท!
