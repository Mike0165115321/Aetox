# 🌌 สถาปัตยกรรมระบบ AetoxClaw: Stateless Core & History Injection
**พิมพ์เขียวการจัดการเอเจนต์แบบ Lightweight**

เอกสารฉบับนี้อธิบายโครงสร้างระบบ AetoxClaw เวอร์ชันปรับปรุงใหม่ (Lightweight Refactor) ซึ่งเน้นความเรียบง่าย ความเร็ว และการประหยัดทรัพยากรเครื่อง

---

## 1. การทำงานภาพรวม (The Stateless Flow)

AetoxClaw เปลี่ยนมาใช้สถาปัตยกรรมแบบ **Stateless Core** ซึ่งหมายความว่าเอเจนต์ (Agents) จะไม่เก็บสถานะหรืองานที่ทำค้างไว้ในตัว แต่จะได้รับข้อมูลที่จำเป็น (Context) ผ่านการส่งต่อ (Injection) จากตัวกลาง (Dispatcher) ในทุกครั้งที่มีการเรียกใช้งาน

### 🗺️ เส้นทางการไหลของข้อมูล (Data Journey)

```text
[ ผู้ใช้ ] ──► [ MAIN AGENT ] ──► [ DISPATCHER ] ──► [ EXECUTOR ] ──► [ CRITIC ]
  (คำสั่ง)      (วางแผนงาน)      (ฉีดประวัติงาน)      (เรียกใช้เครื่องมือ)   (ตรวจผลลัพธ์)
                                       ▲                                 │
                                       └─────────────────────────────────┘
                                            (วนซ้ำ/แก้ไขหากพลาด)
```

## 2. ระบบจัดการความจำแบบใหม่ (Smart History Injection)

เพื่อลดภาระการประมวลผล (CPU/RAM) เราได้ยกเลิกระบบ RAG และการสรุปเนื้อหาอัตโนมัติออก แล้วเปลี่ยนมาใช้การเลือกส่ง "ประวัติ" ที่จำเป็นตามประเภทของงาน:

### 💬 โหมดแชท (Chat Mode)
*   **วิธีการ:** ส่งประวัติการคุยย้อนหลัง **3-5 ข้อความล่าสุด**
*   **เหตุผล:** เพื่อให้โมเดลเข้าใจบริบทการสนทนาปัจจุบันได้โดยไม่ต้องอ่านประวัติทั้งหมดตั้งแต่เริ่ม ซึ่งช่วยให้ประมวลผลได้เร็วขึ้นมาก

### 📝 โหมดทำงานตามแผน (Plan Mode)
*   **วิธีการ:** ส่งเฉพาะข้อมูลที่จำเป็นต่อการทำงานต่อเนื่อง
    *   **ผลลัพธ์ขั้นตอนล่าสุด:** เพื่อให้รู้ว่าเพิ่งทำอะไรสำเร็จไป
    *   **สรุปแผนงานปัจจุบัน:** เพื่อให้รู้ว่าตอนนี้อยู่ขั้นตอนไหนของแผนใหญ่
*   **เหตุผล:** ป้องกันการส่งข้อมูลซ้ำซ้อนเกินความจำเป็น (Context Bloat)

---

## 3. ส่วนประกอบสำคัญ (Core Components)

1.  **Main Agent (ผู้วางแผน):** รับคำสั่งและแตกงานเป็นขั้นตอน (JSON)
2.  **Dispatcher (ผู้ควบคุม):** หัวใจหลักในการรันงานแบบ Async และเป็นผู้จัดการการฉีดประวัติ (History Injection)
3.  **Executor Agent (ผู้ลงมือ):** รับคำสั่งพร้อมบริบทที่ถูกฉีดเข้ามา แล้วเรียกใช้ Tools
4.  **Critic Agent (ผู้ตรวจสอบ):** ให้คะแนนและคำแนะนำหากงานไม่สำเร็จตามเป้าหมาย
5.  **Session Context (ความจำชั่วคราว):** เก็บข้อมูลประวัติแบบ Plain List ใน RAM เพื่อความรวดเร็วสูงสุด

---

## 4. มาตรฐานความปลอดภัย (Safety)

*   **Path Sandbox:** จำกัดการทำงานของเอเจนต์ให้อยู่ในโฟลเดอร์ที่กำหนดเท่านั้น
*   **Permission Manager:** ระบบขออนุมัติก่อนทำกิจกรรมที่มีความเสี่ยง (เช่น ลบไฟล์)
*   **Async Timeout:** ระบบตัดการทำงานหากเอเจนต์รันงานค้างนานเกินไป
